# 📘 Week 4: Computer Fundamentals (For Cybersecurity)

## 🧩 Part 2: Processes, Programs & How Apps Run

---

### 🧠 Why This Matters?

In Week 3, we learned how a computer works...

Now the question is:

👉 What actually happens when you open an app?

Apps don’t open magically.

👉 They are loaded, executed, and run inside the system.

If you understand this → you understand how systems behave.

---

### 💻 Program vs Process

👉 Program = File stored in your system  
👉 Process = Program that is currently running  

📌 Example:
- Chrome installed → Program  
- Chrome opened → Process  

👉 Program is stored → Process is active  

---

### ⚙️ What Happens Inside?

When you open an app:

👉 It loads from storage  
👉 Moves into RAM  
👉 CPU starts executing it  
👉 A process is created  

👉 Multiple processes can run at the same time  

---

### 🧵 Threads

👉 Thread = Small task inside a process  

📌 Example:
- Load webpage  
- Handle input  
- Play video  

👉 More threads = smoother performance  

---

### 🔗 Simple Analogy

Think of it like this:

👉 Program → Recipe 📖  
👉 Process → Cooking 🍳  
👉 Threads → Different chefs 👨‍🍳  

👉 More chefs = faster work  

---

### ⚠️ Why This Matters in Cybersecurity?

👉 Harmful programs can run like normal apps  
👉 Some apps run silently in background  
👉 Some programs may access your data  

📌 Signs to notice:
- Unknown app running  
- High CPU usage  
- Unusual background activity  

👉 If you understand processes → you can detect unusual behavior  

---

### 🚨 Security Insight

👉 Not every running process is safe  

👉 Monitor running applications  
👉 Check system behavior  
👉 Look for unusual activity  

---

### 💡 Key Takeaway

👉 Computers convert programs into processes  

🔥 If you understand how things run,  
you understand how things behave
